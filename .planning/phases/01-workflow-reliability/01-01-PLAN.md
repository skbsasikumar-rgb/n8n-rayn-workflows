<objective>
Set Railway environment variables on both n8n and NocoDB services to prevent concurrent trigger overlaps, raise payload size limits, extend task runner timeout, and remove the silent NocoDB row cap that truncates query results.

Purpose: These env vars are the foundation for all subsequent workflow fixes. N8N_CONCURRENCY_PRODUCTION_LIMIT=1 is secondary defense for the race condition fix (FIX-01). DB_QUERY_LIMIT_MAX=100000 removes the silent row cap that makes pagination fixes (FIX-06, FIX-07) effective. Without these, workflow JSON changes alone cannot fix the pipeline.

Output: Railway env vars confirmed set on both services; services redeployed.
</objective>

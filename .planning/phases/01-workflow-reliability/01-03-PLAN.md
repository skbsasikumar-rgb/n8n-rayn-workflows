Insert a Wait node (type: `n8n-nodes-base.wait`, mode: `timeInterval`) before EACH OpenRouter HTTP Request node in the enrichment chain inside the Loop Over Items body.

From RESEARCH.md, the OpenRouter nodes in wf-latest are:
1. `OpenRouter - Clean Name` -- cleans company name
2. `OpenRouter - URL Dedup` -- validates/deduplicates URLs
3. `OpenRouter - HIA Classification` (or similar name) -- classifies HIA relevance
4. `OpenRouter - Email Generation` (or similar name) -- generates enrichment emails

For each OpenRouter node, add a Wait node BEFORE it:
- Wait node name: `Wait Before {OpenRouter Node Name}` (e.g., "Wait Before OpenRouter - Clean Name")
- Wait node type: `n8n-nodes-base.wait`
- Wait node parameters:
  ```json
  {
    "resume": "timeInterval",
    "interval": 1,
    "unit": "seconds"
  }
  ```

Wiring for each Wait node:
- The node that previously connected TO the OpenRouter node now connects TO the Wait node
- The Wait node output connects TO the OpenRouter node
- The OpenRouter node's output connections remain unchanged

IMPORTANT: Use n8n's built-in Wait node, NOT a Code node with `sleep()` or `setTimeout()`. Code node timing functions count against the 300-second task runner timeout (`N8N_RUNNERS_TASK_TIMEOUT`). The Wait node pauses execution and resumes without consuming task runner time.

Per D-09, the specified backoff pattern is 1s -> 2s -> 5s -> 13s. For Phase 1 simplicity (Claude's Discretion on exact timing), use a fixed 1-second Wait before each of the 4 OpenRouter calls. This adds 4 seconds per lead (20 seconds total for 5 leads) but eliminates 429 risk at 5-lead batch volume. Escalating backoff can be added in a future phase if needed.

Each Wait node needs a unique ID (generate a UUID) and unique position coordinates in the workflow JSON (offset from the OpenRouter node it precedes, e.g., x - 200 for horizontal spacing).

4. **Remaining `.first()` references audit:** Scan ALL nodes inside the loop body for any remaining `.first()` references that were not caught in Plan 02. Every node inside the loop must use `$json.fieldName` for current-item data, not `$('SomeNode').first().json.fieldName`.

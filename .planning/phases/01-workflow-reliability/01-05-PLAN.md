The workflow structure:

```
Schedule Trigger (every 30 min)

GET Timeout Leads (HTTP GET NocoDB ?where=(status,eq,verification_timeout)&limit=50)
  |
Parse Timeout Leads (Code node: extract list, check if empty)
  |
IF No Leads -> NoOp (stop)
  |
Loop Over Items [Batch Size: 1]
  | (loop body)
  |
No2Bounce Submit (HTTP POST to No2Bounce API to re-submit email for validation)
  |
Wait 15s (Wait node: 15 seconds for No2Bounce to process)
  |
No2Bounce Poll (HTTP GET No2Bounce tracking endpoint)
  |
Parse Poll Result (Code node: extract isReady, isValid)
  |
PATCH NocoDB Status (HTTP PATCH: write valid/invalid/verification_timeout based on result)
  |
  back to Loop Over Items
  | (done output)
NoOp (end)
```

Code node (runOnceForAllItems) for Read Discovery:
```javascript
const baseUrl = 'https://nocodb-production-f802.up.railway.app';
const projectId = 'pb7f1zou786xyqc';
const tableId = 'mp36f8mgk115qse'; // discovery table
const token = 'SqqM8YcDuzXGijg0oY7PvqTcPX5H-r2sYqbAOWTN';
const pageSize = 1000;

let page = 1;
let allRecords = [];
let isLastPage = false;

while (!isLastPage) {
  const url = `${baseUrl}/api/v1/db/data/noco/${projectId}/${tableId}` +
    `?limit=${pageSize}&page=${page}`;
  const response = await $http.request({
    method: 'GET',
    url,
    headers: { 'xc-token': token }
  });
  allRecords = allRecords.concat(response.list || []);
  isLastPage = response.pageInfo?.isLastPage ?? true;
  page += 1;
}

return allRecords.map(r => ({ json: r }));
```

The existing `Dedup Against Leads` Code node in wf-discovery deduplicates by `company_name` (lowercased). Per RESEARCH.md Pitfall 6, this should use `website_url` as the dedup key.

Find the `Dedup Against Leads` Code node and update its dedup logic:

Before (current -- name-based):
```javascript
// Current pattern: dedup by company_name
const seen = new Set(leadsRows.map(r => String(r.company_name || '').toLowerCase().trim()));
const unique = discoveryRows.filter(r => {
  const name = String(r.company_name || '').toLowerCase().trim();
  if (seen.has(name)) return false;
  seen.add(name);
  return true;
});
```

After (fixed -- URL-based):
```javascript
// Dedup by website_url (canonical dedup key)
const existingUrls = new Set(
  leadsRows.map(r => String(r.website_url || r.url || '').toLowerCase().trim()).filter(u => u)
);
const batchUrls = new Set();
const unique = discoveryRows.filter(r => {
  const url = String(r.website_url || r.url || '').toLowerCase().trim();
  if (!url || existingUrls.has(url) || batchUrls.has(url)) return false;
  batchUrls.add(url);
  return true;
});
```

The `batchUrls` Set handles in-batch dedup (same URL discovered from two area/category combos in the same weekly run). The `existingUrls` Set handles cross-batch dedup against leads already in NocoDB.
